01두 가지 상태: 커서 vs. 선택

Selecto는 사진에 대해 서로 독립적인 두 가지 상태를 구분합니다. 이 구분을 이해하는 것이 선택 동작 전체의 출발점입니다.

selectedIndex

커서(focus)

키보드 포커스가 놓인 단 하나의 사진. 방향키로 이동하며, 그리드에서 테두리로 강조됩니다. 디테일 뷰가 보여주는 사진도 이 커서가 가리키는 사진입니다. "지금 보고 있는 위치"일 뿐, 선택과는 무관합니다.

picked

선택(pick)

사용자가 의도적으로 "고른" 사진들의 집합. 여러 장이 될 수 있고, 체크 표시로 강조됩니다. 이후 BestShots 폴더로 이동되는 대상이 바로 이 집합입니다.

핵심: 커서를 옮긴다고 선택이 바뀌지 않으며, 사진을 선택해도 커서는 그 자리에 머뭅니다. 덕분에 "커서는 그대로 둔 채 마우스 우클릭으로 다른 사진만 콕 집어 선택"하는 흐름이 가능합니다.

02선택 상태 모델

상태는 불변(immutable) 스냅샷인 GalleryState에 담깁니다. lib/features/gallery/gallery_state.dart

class GalleryState {
  final List<Photo> photos;           // 현재 디렉토리에서 스캔된 사진들
  final int selectedIndex;            // 커서 위치 (한 개)
  final Set<String> picked;           // 선택된 사진들의 "경로" 집합
  final Map<String, AnalysisResult> results; // cacheKey → AI 분석 결과
  // ...
}
  • 선택의 식별자는 Photo.path (절대 경로)입니다. 인덱스가 아닌 경로로 저장하므로, 목록이 재정렬되거나 일부가 제거돼도 선택이 엉뚱한 사진으로 옮겨가지 않습니다.
  • picked는 Set이라 중복이 없고, 포함 여부 확인이 O(1)입니다. 그리드의 각 타일은 picked.contains(photo.path)만 구독하므로, 한 장을 선택해도 그 타일 하나만 리빌드됩니다.
  • 분석 결과 results는 Photo.cacheKey(경로::수정시각::바이트수)로 색인됩니다. 자동 선별은 이 점수를 사용합니다.

03수동 선택 동작

모든 수동 선택은 GalleryController의 메서드로 수렴합니다. lib/features/gallery/gallery_controller.dart

메서드동작
togglePickCurrent() 커서가 가리키는 사진의 선택을 토글. Space 및 디테일 뷰에서 호출됩니다.
togglePickByPath(path) 커서와 무관하게 특정 사진을 토글. 이미 선택돼 있으면 해제, 아니면 추가. 그리드/필름스트립/디테일 뷰의 우클릭이 이걸 부릅니다.
pickAll() 현재 목록의 모든 사진을 선택. Ctrl/⌘ + A
unpickAll() 선택 전체 해제 (picked를 비움). Ctrl/⌘ + D

토글의 핵심 구현은 Set의 add 반환값을 이용한 단 두 줄입니다.

void togglePickByPath(String path) {
  final picked = {...state.picked};
  if (!picked.add(path)) picked.remove(path); // 추가 실패(=이미 있음) → 제거
  state = state.copyWith(picked: picked);
}
우클릭으로 선택하는 흐름은 커서를 이동시키지 않습니다. 즉 키보드로 사진을 훑어보면서(←→) 동시에 마우스로 마음에 드는 컷만 우클릭해 모아두는 작업이 자연스럽게 됩니다.

04자동 선별 — Select Best Shots

우상단 별(★) 버튼이 자동 선별입니다. AI 분석 점수를 기준으로 상위권 사진을 한 번에 선택에 추가합니다. (분석 결과가 하나도 없으면 버튼은 비활성화됩니다.)

// gallery_controller.dart
void selectBest({double topPercentile = 0.2}) {
  final selector = ref.read(selectBestShotsProvider);
  final best = selector(
    photos: state.photos,
    resultsByCacheKey: state.results,
    topPercentile: topPercentile, // 기본 상위 20%
  );
  state = state.copyWith(
    picked: {...state.picked, ...best.map((p) => p.path)}, // 기존 선택에 "추가"
  );
}
  • 덮어쓰지 않고 합칩니다(additive). 손으로 골라둔 선택은 유지한 채, 자동 선별 결과를 Set 합집합으로 더합니다. 완전히 새로 시작하려면 먼저 Ctrl/⌘ + D로 비우면 됩니다.
  • 점수의 출처는 활성 모델의 qualityScore(0~10)입니다. NIMA 계열은 MOS(1~10), MANIQA 계열은 시그모이드 출력×10입니다. 모델을 바꾸면 인메모리 결과가 초기화되므로, 자동 선별은 항상 현재 모델 기준입니다.

05선별 알고리즘과 계산 예시

실제 선별 로직은 순수 도메인 코드인 SelectBestShots에 있습니다(Flutter/IO 의존 없음, 그래서 단위 테스트가 쉽습니다). lib/domain/usecases/select_best_shots.dart

단계

  1. 분석된 사진만 추리기 — results에 항목이 있는 사진만 후보. 분석되지 않은 사진은 애초에 제외됩니다.
  2. 점수 오름차순 정렬 — 후보들의 qualityScore를 정렬.
  3. 임계값(threshold) 산출 — (1 − topPercentile) 분위수 위치의 점수를 임계값으로 삼습니다. 
    cutoffIndex = floor(N × (1 − topPercentile)).clamp(0, N−1)
threshold   = scoresAsc[cutoffIndex]
  4. 임계값 이상 모두 선택 — qualityScore >= threshold인 사진을 전부 반환.
경계의 동점(tie)은 모두 포함합니다. 임계값과 같은 점수가 여럿이면 결과 집합이 topPercentile × N보다 약간 커질 수 있는데, 이는 임의의 동점 탈락보다 바람직하다고 보아 의도된 동작입니다.

예시 — 10장, 상위 20%

점수를 오름차순 정렬했다고 가정합니다. N = 10, topPercentile = 0.2 → cutoffIndex = floor(10 × 0.8) = 8 → 인덱스 8의 점수가 임계값.

2.1 3.4 4.0 5.2 5.9 6.3 7.1 7.8 8.4 ▲ 9.2

인덱스: 0…9 / ▲ = cutoffIndex 8, threshold = 8.4 / 초록 = 선택됨(≥ 8.4) → 최종 2장 선택 (8.4, 9.2).

만약 인덱스 7의 점수도 8.4였다면(동점) 3장이 선택됩니다. 분석된 사진이 0장이면 빈 결과를 반환하고 선택은 변하지 않습니다.

06선택본 이동 — Move to BestShots

별 버튼 오른쪽의 이동 버튼은 선택된 사진들을 현재 디렉토리 아래 BestShots 폴더로 물리적으로 옮깁니다(폴더가 없으면 생성). 선택이 비어 있으면 비활성화됩니다.

// gallery_controller.dart — 요지
Future<int> moveSelectedToBestShots() async {
  final destDir = p.join(state.rootPath!, 'BestShots');
  for (final photo in /* picked 에 속한 사진들 */) {
    await repo.movePhoto(photo, destDir); // 같은 볼륨이면 rename, 다르면 copy+delete
  }
  // 이동된 사진을 photos/picked/results 에서 제거하고 커서를 보정
}
  • 파일 이동은 데이터 계층의 PhotoRepository.movePhoto가 담당합니다. 같은 드라이브면 원자적 rename, 드라이브가 다르면 copy 후 delete로 폴백합니다.
  • 이름 충돌 시 이름 (1).jpg, 이름 (2).jpg 식으로 자동 회피합니다.
  • 이동에 성공한 사진은 인메모리 상태(photos, picked, results)에서 제거되고, 커서 인덱스는 남은 범위로 클램프됩니다. 완료 시 "Moved N photos to BestShots" 스낵바를 띄웁니다.
  • 일부만 실패하면 성공분은 그대로 반영하고, 실패는 에러 채널로 표면화됩니다.
디렉토리 스캐너는 재귀적이라, 폴더를 다시 열면 BestShots 안의 사진도 다시 스캔되어 나타납니다. 현재는 이동 직후 인메모리 목록에서만 감추는 방식입니다.

07단축키 · 마우스 동작

입력은 RawKeyboardListener 대신 Shortcuts/Actions로 구현되어 포커스 트래버설을 존중합니다. lib/features/gallery/widgets/gallery_shortcuts.dart

← →커서 이동(±1)커서

08데이터 흐름과 아키텍처

선택은 UI → 컨트롤러 → (자동 선별의 경우) 도메인 유스케이스로 흐릅니다. UI는 절대 데이터/도메인 내부를 건너뛰지 않습니다.

[자동 선별 ★] GalleryScreen 별 버튼 │ ctrl.selectBest() ▼ GalleryController ──read──▶ selectBestShotsProvider │ │ │ ▼ │ SelectBestShots(use case) ← 순수 도메인, 점수 분위수 계산 │ │ List<Photo> (상위권) ◀─────────────────────────┘ │ picked = picked ∪ best.paths ▼ GalleryState.copyWith(picked: …) ──▶ 그리드 타일 중 해당 path 만 리빌드 [수동 선택] Space / 우클릭 / Ctrl+A / Ctrl+D ▼ GalleryController.toggle… / pickAll / unpickAll ▼ GalleryState.picked 갱신 [선택본 이동] 이동 버튼 ▶ ctrl.moveSelectedToBestShots() ▼ PhotoRepository.movePhoto() (rename / copy+delete) ▼ photos·picked·results 에서 제거 + 스낵바
  • 상태 관리: Riverpod @riverpod 코드젠 기반의 GalleryController가 단일 진실 공급원입니다.
  • 세분화된 구독: 각 타일이 자기 path의 picked 포함 여부만 select로 구독하므로, 수천 장 규모에서도 선택 토글이 전체 그리드를 리빌드하지 않습니다.
  • 순수 도메인 분리: 선별 알고리즘은 lib/domain/의 순수 함수라 위젯 없이 테스트됩니다(test/domain/select_best_shots_test.dart).

09경계 조건과 주의점

상황동작
분석 결과가 0개별 버튼 비활성화. 호출돼도 SelectBestShots가 빈 목록 반환 → 선택 불변.
선택이 0개일 때 이동이동 버튼 비활성화. moveSelectedToBestShots는 0을 반환하고 아무것도 하지 않음.
모델 전환인메모리 results가 초기화되고 진행 중 분석은 취소됨. 자동 선별은 다시 분석해야 가능. (수동 picked는 유지)
파일 이동 중 일부 실패성공분만 상태에서 제거하고 반영, 실패는 에러로 표면화.
이동 후 폴더 재오픈스캐너가 재귀적이라 BestShots 내부 사진이 다시 목록에 나타남.
분석되지 않은 사진자동 선별 후보에서 제외(점수가 없으므로). 수동 선택은 가능.
